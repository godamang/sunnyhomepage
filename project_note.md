## 써니코딩 AI 교육 철학

### 핵심 메시지

> AI가 코딩하는 시대, 코딩을 아는 아이가 앞서갑니다.

- Claude, Codex 등 AI가 코딩을 매우 잘한다는 점은 인정한다.
- 하지만 AI에게 정확하게 지시하고 결과를 검증하려면 코딩 기본지식이 필요하다.
- 데이터 누락, 잘못된 테이블 조인, 비효율적인 코드, 존재하지 않는 데이터를 채운 백테스트처럼 겉으로 드러나지 않는 오류가 생길 수 있다.
- 작은 프로젝트보다 프로젝트가 커질수록 AI의 실수와 한계가 커질 수 있다.
- 결과물의 겉모양이 완벽해도 내부 논리와 데이터가 정확한지는 사람이 확인해야 한다.
- 따라서 아이들에게 바이브 코딩부터 가르치지 않는다.
- 먼저 읽고 생각하고 문제를 해결하는 힘과 코딩 기본기를 갖추게 한다.

### 학년별 방향

- 초등 3~4학년: 독서·문해력과 코딩 기본기, 스스로 생각하는 습관
- 초등 5~6학년: 독서와 코딩, 데이터 가공과 AI 이전 단계의 기본원리
- 중등 이후: 충분한 기본기를 바탕으로 AI 활용 및 결과 검증 프로젝트

### 써니코딩 소개에 반영한 내용

- 원장은 교육학·컴퓨터공학 전공 및 개발 실무 경험을 갖고 있다.
- 원장이 직접 AI와 데이터 프로젝트를 개발하며 기본기와 검증 능력의 중요성을 확인했다.
- 아이가 AI의 결과를 그대로 믿지 않고 판단하고 검증할 수 있도록 지도한다.

## 과정별 콘텐츠 기준

### 1. 정규 코딩교육

- 원장이 작성한 기존 `curriculum.html` 커리큘럼을 그대로 사용한다.
- 학년만으로 반을 정하지 않고 경험, 이해도, 관심과 진로 목표에 따라 시작점을 정한다.
- 블록코딩에서 파이썬·C·Java·AI·데이터·웹·게임·로봇 프로젝트로 확장한다.
- 상단 메뉴는 다른 페이지보다 작고 간결하게 유지한다.
- 상단에 눈에 띄는 `써니코딩 홈` 버튼을 둔다.

### 2. 수행평가·세특반

- 과제 대행이나 세특 문구 작성으로 표현하지 않는다.
- 학생이 문제를 분석하고 알고리즘을 설계하고 직접 구현하도록 지도한다.
- 오류 수정, 테스트, 보고서와 발표까지 학생이 설명할 수 있도록 한다.
- 은평구 중·고등학교별 공식 평가계획을 조사해 학교별로 안내한다.
- 학교별 항목은 `학교 / 학년 / 과목 / 수행평가 유형 / 반영 비율 / 평가 시기 / 써니코딩 준비 방법`으로 정리한다.
- 공식 평가계획이 확인되지 않은 학교의 내용을 추측해서 작성하지 않는다.

우선 조사할 학교:

- 중학교: 은평중, 불광중, 연천중, 신도중, 대성중, 덕산중, 선정중, 충암중, 숭실중, 예일중, 영락중, 명지중
- 고등학교: 신도고, 대성고, 선정고, 충암고, 숭실고, 예일여고 등

### 3. 서울시 SW영재원

- 현재 써니코딩에서는 선린인터넷고등학교가 운영하는 서울시교육청 SW영재원만 진행한다.
- 다른 대학부설 영재원을 일반적으로 소개하지 않는다.
- 2026학년도 공식 자료 기준:
  - 신규 중2 SW 과정: 40명 이내
  - 중3 SW 과정: 전년도 수료예정자 대상 30명 이내
  - GED에서 학생 지원 후 담임교사 추천 요청
  - 영재성, 비판적 사고력, 종합적 탐구능력과 문제 해결 과정 중심
- 모집 학년·인원·일정·평가 방식은 매년 공식 모집요강으로 갱신한다.
- 써니코딩 수업은 지원 자격 확인, 프로그래밍 기본기, 창의적 문제해결, 서술·설명, 면접·인성 준비로 구성한다.

### 4. 특성화고 진학반

입시 정보는 허브 페이지와 학교별 상세 페이지로 분리한다.

- `admission.html`: 특성화고 진학반 허브
- `admission-semyung.html`: 세명컴퓨터고
- `admission-mirim.html`: 미림마이스터고
- `admission-sunrin.html`: 선린인터넷고
- `admission-hansei.html`: 한세사이버보안고

학교 정보 기준:

- 세명컴퓨터고: 특성화고
- 선린인터넷고: 공립 남녀공학 특성화고
- 한세사이버보안고: 남녀공학 특성화고, 사이버보안 중심
- 미림마이스터고: 과거 미림여자정보과학고였으나 2024년 남녀공학으로 전환
- 미림은 마이스터고이므로 나머지 학교와 모집 시기·전형 체계가 다르다.

페이지 정보는 우선 2026학년도 모집요강을 기준으로 작성하고, 2026년 10월경 다음 학년도 공식 요강으로 갱신한다.

써니코딩이 돕는 내용:

- 학교·학과 적합성 분석
- 학생별 프로그래밍과 전공 기본기
- 학생 주도 프로젝트
- 개발 과정과 포트폴리오 정리
- 실제 경험과 지원동기·진로계획 연결
- 프로젝트 원리와 문제 해결 과정에 대한 심층면접 연습

## 2026-07-18 작업 기록

### 완료한 작업

- 메인과 교육과정 페이지에 남아 있던 `{{$template}}/img/main/...` 이미지 경로를 실제 `image/...` 경로로 변경했다.
- 존재하지 않는 `icon3.ico`, `og-image.jpg` 참조를 저장소에 있는 파비콘과 수업 이미지로 교체했다.
- 메인, 교육과정, 수행평가·세특, SW영재원, 특성화고 허브와 학교별 상세 페이지의 SEO 제목·설명·canonical·Open Graph·Twitter 카드 정보를 보강했다.
- 메인 페이지의 LocalBusiness 구조화 데이터 이미지 경로를 실제 이미지로 수정했다.
- 교육과정 페이지의 상단 메뉴 높이·로고·간격을 줄이고 메뉴명을 `홈 / 교육과정 / 수행평가 / 영재원 / 진학 / 상담`으로 간결하게 바꿨다.
- 교육과정 페이지의 내부 목차를 `학습 로드맵 / 초등 과정 / 중·고등 과정 / 자주 묻는 질문` 네 항목으로 줄였다.
- 로컬 확인용 `preview_server.py`를 추가했다. 호스팅용 HTML 문자 선언은 변경하지 않고 로컬 미리보기 응답에만 UTF-8 헤더를 사용한다.

### 호스팅 문자 처리 주의사항

- 현재 이용 중인 호스팅 서비스에서는 메인·교육과정 페이지의 아래 선언을 유지해야 한다.
- 해당 부분을 일반적인 HTML5 UTF-8 선언으로 임의 변경하지 않는다.

```html
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="ko" lang="ko">
<meta charset="ko" />
```

### 은평구 학교별 정보 수행평가 조사 현황

- 은평중학교 공식 홈페이지에 `평가계획` 메뉴가 존재하는 것을 확인했다.
- 해당 메뉴는 외부 사용자에게 `게시판 접근 권한이 없습니다`라고 표시되어 세부 평가계획을 확인할 수 없었다.
- 일반 검색에 노출되는 과외·학원 글은 학교별 정보 과목의 평가 유형과 반영 비율을 뒷받침할 공식 근거가 부족했다.
- 학교별 표에는 공식 평가계획, 가정통신문 또는 학생에게 배부된 평가 안내문으로 확인된 내용만 넣는다.
- 자료가 비공개인 학교는 추측하지 않고 `공식 자료 확인 중` 또는 기준 학기와 확인 상태를 명시한다.
- 향후 학원 재학생이 받은 평가계획표·수행평가 안내문을 개인정보 제거 후 확인 자료로 활용하는 방법을 검토한다.

## 2026-07-20 작업 기록

### 화면과 메뉴 수정

- 메인 페이지 원장 사진을 기존보다 세로로 길게 표시하도록 비율을 `3:4`로 조정했다.

- `school-record.html` 상단 메뉴에 메인 페이지로 이동하는 `홈` 항목을 추가했다.

### 특성화고 입시 페이지 개편

- 지원 학교 순서를 선린인터넷고, 한국디지털미디어고, 세명컴퓨터고, 미림마이스터고, 한세사이버보안고 순으로 정리했다.
